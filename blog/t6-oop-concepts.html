Blocks, Procs, and Lambdas

Blocks, Procs, and Lambdas are small chunks of code.

(1..3).each { |i| p i} # The block is between the curly brackets

p = Proc.new { |i| p i} # The Proc is the bit between the curly brackets, assigned to an object called "p"
(1..3).each(&p) # The ampersand tells Ruby to use the Proc "p"

l = lambda { |i| p i} # Lambda syntax is similar to Proc syntax, but slightly different
(1..3).each(&l)

As you may have inferred from the naming of the Proc, short for Procedure, and lambda, Procs and lambdas are objects, which can be reused. Procs, lambdas, and blocks are actually all forms of Procs, but blocks are not re-usable (you'd have to re-write the whole block), while Procs and lambdas are (just reference the &Proc or &lambda again). You can also call procs and lambdas with the method :call. 

p = Proc.new { puts "hi" }
p.call # outputs "hi"

l = lambda { puts "hi" }
l.call #outputs "hi"

Blocks, Procs, and lambdas can all also be used as arguments for a method, but a method can only have one Block as a argument, while the number of Procs or lambdas as arguments is not limited. 

def method (proc, proc2, proc3)
end

Procs vs Lambdas

Procs differ from lambdas in a few small ways. If you pass them the wrong number of arguments, a Proc will still run or return nil, while a lambda will raise an error.

p = Proc.new { |i| p i }
p.call(1) # outputs 1
p.call # outputs nil
p.call(1, 2, 3) #outputs 1

l = lambda { |i| p i }
l.call(1) # outputs 1
l.call #outputs ArgumentError: wrong number of arguments (0 for 1)
l.call(1, 2, 3) # outputs ArgumentError: wrong number of arguments (3 for 1)

Procs and lambdas handle "return" slightly differently. If you return from a lambda, it will exit the lambda call, but not the method you are in. If you return from a Proc, it will exit the Proc call and the method.

p = Proc.new { return }
def proc_method
    p.call
    puts "We're still doing the code inside the method!"
end # proc_method will not print out "We're still doing the code inside the method!". As soon as the Proc does "return", the computer will leave the method.

l = lambda { return }
def lambda_method
    l.call
    puts "We're still doing the code inside the method!"
end # lambda_method will print out "We're still doing the code inside the method!". When the lambda does "return", it only exits the lambda call and then continues doing the next line in the method.
